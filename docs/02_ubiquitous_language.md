# ユビキタス言語（Ubiquitous Language）

## 1. ユビキタス言語とは

**ユビキタス言語** とは、開発チームとドメインエキスパート（ビジネス担当者）が
**同じ意味で共有する言葉** のことです。

「ユビキタス（Ubiquitous）」= 「どこにでも存在する」

つまり、**会話・ドキュメント・コード・テスト** のすべてで同じ言語を使うということです。

---

## 2. なぜユビキタス言語が重要か

### 悪い例（言葉がバラバラ）

```
営業担当者：「顧客がカートに商品を入れて購入確定した」
開発者のコード：user.addItem(product); basket.checkout();
DB のテーブル：tbl_user_cart, order_master
```

- `顧客` = `user`？ `customer`？
- `カート` = `basket`？ `cart`？
- `購入確定` = `checkout`？ `purchase`？ `order`？

**ビジネスとコードの間に翻訳コストが発生し、認識ズレがバグを生む。**

### 良い例（ユビキタス言語で統一）

```
営業担当者：「顧客（Customer）が注文（Order）を発注した（Place）」
開発者のコード：customer.placeOrder(orderItems);
DB のテーブル：customers, orders
テスト：顧客が注文を発注する
```

## 3. EC サイトのユビキタス言語辞書

このプロジェクトで使う言葉を定義します。

### 👤 顧客（Customer）
- EC サイトに登録し、商品を購入できるユーザー
- **顧客 ID（CustomerId）** で一意に識別される
- 同義語として「ユーザー」は使わない

### 📦 注文（Order）
- 顧客が商品を購入する意思を示した取引の単位
- **注文 ID（OrderId）** で一意に識別される
- 1 つの注文には 1 つ以上の **注文明細（OrderItem）** が含まれる
- 「カゴ」「トランザクション」は使わない

### 📋 注文明細（OrderItem）
- 注文の中の個々の商品行
- 注文 ID・商品・数量・単価で構成される

### 🏷️ 注文ステータス（OrderStatus）
- 注文の現在の状態を表す
- `PENDING`（注文受付）→ `CONFIRMED`（注文確定）→ `SHIPPED`（発送済）→ `DELIVERED`（配達完了）→ `CANCELLED`（キャンセル）

### 💰 金額（Money）
- 数値と通貨コードの組み合わせ
- 例：`1,500 JPY`
- 単なる数値（`1500`）とは異なる概念

### 🛒 発注する（Place an Order）
- 顧客が注文を確定させるアクション
- 「購入する」「チェックアウトする」は使わない

### ✅ 注文を確定する（Confirm an Order）
- 店側が注文を受け付け確定するアクション

### ❌ 注文をキャンセルする（Cancel an Order）
- 注文を取り消すアクション
- SHIPPED 以降はキャンセル不可

## 4. ユビキタス言語をコードに反映する

ユビキタス言語はそのままクラス名・メソッド名に使います。

```typescript
// ❌ 悪い例：ビジネス用語とコードが乖離している
class UserManager {
  submitPurchase(userId: string, items: CartItem[]): void { ... }
}

// ✅ 良い例：ユビキタス言語がそのままコードになっている
class Order {
  static place(customer: Customer, items: OrderItem[]): Order { ... }
  confirm(): void { ... }
  cancel(): void { ... }
}
```

---

## 5. 言語の発展と更新

ユビキタス言語は**一度決めたら終わりではありません。**

- ドメインエキスパートとの会話を通じて継続的に洗練される
- 新しい概念が生まれたら辞書に追加する
- 言葉の意味が変わったら、コードも同時に更新する

> **コードはドキュメントである。コードを読めばビジネスが理解できる状態が理想。**

---

## 6. まとめ

| ポイント | 説明 |
|---------|------|
| 共通言語 | チーム全員が同じ意味で使う言葉を定義する |
| コードへの反映 | クラス名・メソッド名・変数名に使う |
| 継続的更新 | 知識の深まりとともに言語も進化させる |
| 翻訳禁止 | ビジネス用語 → 技術用語への翻訳レイヤーを作らない |
