# 戦略的設計（Strategic Design）

## 1. 境界づけられたコンテキスト（Bounded Context）

大きなシステムを 1 つのドメインモデルで表現しようとすると、
同じ言葉が異なる意味を持つ問題が起きます。

### 問題：「顧客（Customer）」は一つか？

```
営業部門の「顧客」：名前・住所・購買履歴が重要
配送部門の「顧客」：住所・電話番号・時間指定が重要
経理部門の「顧客」：請求先・支払い方法・与信が重要
```

同じ「顧客」でも **文脈（コンテキスト）によって必要な情報が異なります。**

### 解決策：境界づけられたコンテキスト

**境界づけられたコンテキスト（Bounded Context）** とは、
特定のドメインモデルが有効な範囲（境界）のことです。

```
┌──────────────────────┐  ┌──────────────────────┐
│  注文コンテキスト      │  │  配送コンテキスト      │
│                      │  │                      │
│  Customer            │  │  Recipient（受取人）  │
│    - id              │  │    - name             │
│    - name            │  │    - address          │
│    - email           │  │    - phone            │
│  Order               │  │  Shipment（配送）      │
│  OrderItem           │  │    - orderId          │
└──────────────────────┘  │    - status           │
                          └──────────────────────┘
```

同じ実世界の「顧客」でも、コンテキストごとに **別のモデル** として表現します。

## 2. コンテキストマップ（Context Map）

複数のコンテキストがどのように連携するかを示す地図です。

```
EC サイトのコンテキストマップ

┌─────────────────┐         ┌─────────────────┐
│  注文コンテキスト │ ──────→ │  配送コンテキスト │
│  (Order)        │  注文確定 │  (Shipping)     │
└─────────────────┘  イベント └─────────────────┘
         │
         │ 在庫確認
         ▼
┌─────────────────┐         ┌─────────────────┐
│  在庫コンテキスト │         │  決済コンテキスト │
│  (Inventory)    │         │  (Payment)      │
└─────────────────┘         └─────────────────┘
```

### コンテキスト間の関係パターン

| パターン | 説明 | 例 |
|---------|------|---|
| **共有カーネル** | 一部モデルを共有 | 注文 ID を複数コンテキストで共有 |
| **顧客-供給者** | 一方が API を提供 | 決済サービスが API を提供 |
| **腐敗防止層** | 外部モデルを変換 | 外部 API のレスポンスを自分のモデルに変換 |
| **公開ホストサービス** | 汎用 API を公開 | REST API / GraphQL |

---

## 3. このプロジェクトのスコープ

本サンプルコードでは **注文コンテキスト（Order Context）** に集中して実装します。

```
注文コンテキストの責務
├── 顧客（Customer）の管理
├── 注文（Order）の発注・確定・キャンセル
└── 注文明細（OrderItem）の管理
```

---

## 4. まとめ

| 概念 | 目的 |
|-----|------|
| 境界づけられたコンテキスト | ドメインモデルが有効な範囲を明確にする |
| コンテキストマップ | コンテキスト間の関係を可視化する |
| ユビキタス言語 | コンテキスト内で一貫した言語を使う |

> **大きな問題は分割せよ。各コンテキストは独立して理解できる単位にすべき。**
